# 迁移指南

将 WZX 多 Agent 交付系统迁移到新环境（新 PC / 新项目 / 新 AI 工具）。

## 第 1 层：用户级（每台 PC 一次）

### 1.1 克隆架构仓库

```bash
git clone <仓库地址> wzx-agents
```

### 1.2 迁移到目标项目

```bash
cd <目标项目>

# 创建目录
mkdir -p .github/agents .github/instructions .memory

# 复制 Agent 定义（wzx- 前缀不会与已有文件冲突）
cp <架构仓库>/agents/wzx-*.md .github/agents/

# 复制行为规则
cp <架构仓库>/instructions/wzx-*.md .github/instructions/

# 初始化项目记忆
cp <架构仓库>/projects/_template/* .memory/

# 如果项目有 copilot-instructions.md，追加规则
if [ -f .github/copilot-instructions.md ]; then
    echo "" >> .github/copilot-instructions.md
    echo "<!-- WZX 多Agent 交付系统 -->" >> .github/copilot-instructions.md
    cat <架构仓库>/instructions/wzx-运行模型.md >> .github/copilot-instructions.md
    echo "<!-- /WZX -->" >> .github/copilot-instructions.md
fi
```

### 1.3 验证迁移

```
告诉 AI："读取 .github/agents/wzx-组合PM.md 和 .memory/交接.md，确认迁移成功"
```

## 第 2 层：项目级（每个项目一次）

### 2.1 填写项目简报

编辑 `.memory/项目简报.md`，填写：
- 技术栈（语言、框架、数据库等）
- 项目范围和目标
- 关键约束和假设

### 2.2 初始化项目状态

编辑 `.memory/项目状态.yaml`，设置：
- `stage: discovery`
- `status: active`
- `active_agent: 组合PM`

### 2.3 创建初始交接

编辑 `.memory/交接.md`：
- 目标：项目初始设定
- 下一步动作：`组合PM 应该 分配第一个任务`

### 2.4 注册到组合索引

如果管理多个项目，编辑架构仓库的 `portfolio/组合索引.md` 添加项目。

## 第 3 层：会话级（自动）

每次开始工作时告诉 AI：

```
读取 adapters/detection.md，检测环境并按适配指南开始工作
```

AI 会自动：
1. 检测当前工具（Code CLI / ZCode / Copilot / 通用）
2. 读取对应的适配指南
3. 读取项目记忆文件
4. 按角色定义执行工作

## 冲突处理

### 目标项目已有 Agent 定义

`wzx-` 前缀自动隔离。已有的 agent 文件不受影响。

### 目标项目已有 Instructions

`wzx-` 前缀自动隔离。已有的 instruction 文件不受影响。

### 跨 PC 路径不一致

项目记忆文件中只使用相对路径。不写绝对路径。

### 多人协作 Git 冲突

- 交接.md 更新前先 git pull
- 项目状态.yaml 每个字段一行，减少冲突面积
- 发生冲突时以最新交接内容为准
