# 运行模型

## 基本规则
- 使用项目记忆文件作为运行时事实来源。
- 在开始重大工作前先阅读项目简报、项目状态、当前上下文和交接文件。
- 保持交接文件简短、明确、以行动为导向。
- 在取得有效进展后更新持久项目记忆。
- 保持角色之间的边界。

## 路径规范
- 项目记忆路径使用相对于项目根目录的相对路径
- 禁止在记忆文件中写入绝对路径
- 项目记忆文件位于 `.memory/` 目录下（迁移到实际项目后）

## 角色边界
- PM 负责优先级和状态卫生。
- Dev 负责实现和环境配置。
- Review/Debug 负责发现问题、识别风险和根因分析。
- Test 负责验证结果和测试缺口记录。
