# 工作区 Agent 运行模型

这个工作区承载一套面向 AI agents 的多项目交付系统。

## 默认运行规则

- 把项目记忆文件作为事实来源，而不是依赖之前的聊天记录。
- 在对项目做实质性工作前，先阅读该项目的简报、状态、当前上下文和交接文件。
- 在结束一次工作会话前，更新项目的当前上下文和交接文件。
- 保持变更最小，并与现有项目状态保持一致。
- PM 负责优先级和状态卫生。
- Dev 负责实现和环境配置。
- Review/Debug 负责发现问题、识别风险和根因分析。
- Test 负责验证结果和测试缺口记录。

## 项目布局

- `组合/` 保存跨项目规划和依赖状态。
- `项目/项目-A` 到 `项目/项目-E` 保存项目级记忆。
- `共享/模板/` 保存可复用模板。
- `.github/agents/` 保存 pilot agent 定义。
- `.github/instructions/` 保存定向 instructions 文件。

## Pilot 工作流

1. PM 读取组合级状态和目标项目状态，然后更新待办列表和项目状态。
2. Dev 读取项目状态，执行实现工作，并更新当前上下文和交接文件。
3. Review/Debug 检查变更或调查故障，并记录发现。
4. Test 验证预期行为并更新测试报告。
5. PM 更新项目状态并决定下一步动作。

## 工具边界概览

| 角色       | 可写范围                                      | 终端权限           |
| ---------- | --------------------------------------------- | ------------------ |
| 组合PM     | 状态文件、待办列表、交接                       | 仅只读查询         |
| 通用开发   | 代码、文档、当前上下文、交接、环境文件         | 开发和构建         |
| 评审调试   | 默认只读，受控修复时可编辑相关文件             | 调试和诊断         |
| 测试工程师 | 测试报告、当前上下文                           | 测试执行           |

## Headless 自动化模式

除手动切换角色外，支持通过编排器自动运行多实例并行工作流。

### 快速使用

```bash
# 启动编排器
bash 脚本/orchestrator.sh start

# 查看状态
bash 脚本/orchestrator.sh status

# 手动添加任务
bash 脚本/orchestrator.sh add-task \
  --project 项目-A --role 通用开发 --type repo \
  --prompt "读取 项目/项目-A/交接.md 的下一步动作，执行实现"

# 停止编排器
bash 脚本/orchestrator.sh stop
```

### 架构

- `orchestrator.sh` — 主入口（启动/停止/状态/添加任务）
- `scheduler.sh` — 调度循环（轮询队列、启动实例、检测完成）
- `instance_mgr.sh` — 实例生命周期（启动/停止/监控）
- `queue.sh` — 任务队列操作（增删改查/交接解析）
- `launch_instance.sh` — 实例启动命令构建
- `monitor.sh` — 健康检查（心跳/超时检测）

### 任务类型

| 类型 | --cwd | 适用场景 |
| ---- | ----- | -------- |
| `repo` | git 仓库目录 | 编码、构建、测试 |
| `ops` | 工作区根目录 | NAS 管理、Docker 配置、系统运维 |

### 模型差异化

| 角色 | 模型 | 原因 |
| ---- | ---- | ---- |
| 组合PM | sonnet | 决策类任务无需最强推理，降低成本 |
| 通用开发 | opus | 编码实现需要强推理 |
| 评审调试 | opus | 根因分析需要强推理 |
| 测试工程师 | haiku | 测试执行模式化，无需强推理 |

### 并发规则

- 同一项目同时只能一个实例工作（项目锁互斥）
- 最多 4 个实例并行运行
- 通用开发最多 2 个并行，其他角色各 1 个
- 实例超时 30 分钟自动终止

### 运行时文件

所有运行时状态保存在 `.run/` 目录下（已加入 .gitignore），包含：
- `orchestrator/` — 编排器状态和日志
- `instances/` — 每个实例的隔离工作目录
- `queue/` — 任务队列（inbox/active/completed/failed）
- `locks/` — 项目级文件锁

### 与手动模式的关系

Headless 模式与手动切换角色完全兼容。编排器运行期间，仍可手动使用交互式 Code 工作在同一工作区。编排器通过项目锁自动避免冲突。
