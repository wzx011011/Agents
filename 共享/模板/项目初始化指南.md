# 项目初始化指南

当需要将一个空的占位项目（如 项目-B）激活为可用状态时，按以下步骤操作。

## 前置条件

- 已确认该项目的真实目标和技术栈。
- 已确认该项目的优先级（由组合PM在组合索引中设定）。

## 步骤

### 步骤 1：填写项目简报

1. 复制 `共享/模板/项目简报模板.md` 到目标项目的 `项目简报.md`。
2. 填写 YAML frontmatter 中的 project_id、name、created、stage。
3. 填写每个章节：使命、范围、非目标、技术栈、约束、关键文件、成功定义。
4. 验证：简报中不包含任何 "替换为" 或 "填写" 占位文本。

### 步骤 2：更新项目状态

1. 打开 `项目状态.yaml`。
2. 更新：summary（一句话概述）、current_goal（当前目标）、next_action（明确的下一步）。
3. 更新 stage 为 `planning`（如果简报已完成但任务未定义）或 `implementation`（如果已有可执行任务）。
4. 移除 blockers 中的 "真实项目细节尚未录入" 条目。
5. 更新 last_updated 为当前日期。

### 步骤 3：创建初始待办列表

1. 复制 `共享/模板/待办列表模板.yaml` 到目标项目的 `待办列表.yaml`。
2. 将示例任务替换为真实任务。
3. 确保至少有 1 个 status=ready 的任务。
4. 确保每个任务有明确的 acceptance_criteria。

### 步骤 4：填写环境说明

1. 如果需要本地环境，填写 `环境/环境搭建.md`。
2. 如果有已知问题，记录到 `环境/已知问题.md`。
3. 更新 `项目状态.yaml` 中的 environment_state（unknown → partial → ready）。

### 步骤 5：更新交接文件

1. 在 `交接.md` 中写入：简报已完成，下一步动作是开始第一个 ready 任务。
2. 更新 active_agent 为即将执行第一个任务的角色。

### 步骤 6：更新组合级文件

1. 更新 `组合/组合路线图.yaml` 中该项目的 next_milestone。
2. 如果该项目有跨项目依赖，更新 `组合/依赖关系.yaml`。

## 验证

初始化完成后，按以下清单验证：

- [ ] `项目简报.md` 中无占位文本
- [ ] `项目状态.yaml` 中 stage 不是 discovery
- [ ] `待办列表.yaml` 中至少有 1 个 status=ready 的任务
- [ ] `交接.md` 中有明确的下一步动作
- [ ] `环境/环境搭建.md` 已填写或标注为不需要
